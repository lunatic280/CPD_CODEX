# 프로젝트 지침

## 프로젝트 배경

이 프로젝트는 다른 학생들이 Codex 사용법을 익히기 위해 진행하는 게시판 앱 프로젝트이다.

앱은 Flutter로 개발하며, 서버 없이 앱 안에서 데이터를 저장할 수 있도록 SQLite를 사용해야 한다.

## 핵심 요구사항

- 클라이언트는 Flutter 앱으로 구현한다.
- 로컬 데이터 저장에는 SQLite를 사용한다. 프로젝트에서 다른 SQLite 라이브러리를 일관되게 채택하지 않는 한 Flutter의 `sqflite` 패키지를 우선 검토한다.
- 게시판의 기본 흐름인 글 목록 조회, 글 상세 조회, 글 작성, 글 수정, 글 삭제를 지원한다.
- 사용자가 명시적으로 요구사항을 바꾸기 전에는 백엔드 서버를 추가하지 않는다.
- Codex를 배우는 학생들이 이해하기 쉽도록 구현과 설명을 단순하고 명확하게 유지한다.

## Markdown 작성 규칙

- 이 프로젝트의 모든 Markdown 파일은 한국어로 작성한다.
- 기존 Markdown 파일을 수정할 때 영어 제목, 표 머리글, 설명 문장이 있으면 한국어로 바꾼다.
- 외부 명령어, 코드 식별자, 패키지명, 파일명은 필요한 경우 원문 그대로 둔다.

## 명령 및 파일 기록

- 이 Codex 대화에서 사용자가 입력한 모든 명령과 요청은 `docs/COMMAND_LOG.md`에 기록한다.
- `사용자 명령 또는 요청` 칸에는 사용자가 입력한 문장을 요약하거나 바꾸지 말고 가능한 한 원문 그대로 적는다.
- 사용자가 여러 줄로 입력한 경우 줄 순서와 내용을 유지하고, Markdown 표 안에서는 `<br>`로 줄바꿈한다.
- 각 명령과 요청마다 다음 내용을 남긴다.
  - 날짜
  - 사용자 명령 또는 요청 원문
  - 수행한 작업
  - 해당 명령으로 생성 또는 수정된 파일
  - 검증 결과 또는 차단 사항
- 파일을 생성하거나 수정한 경우 같은 턴 안에서 기록 파일도 함께 갱신한다.

## 개발 메모

- 학생들이 검토하기 쉽도록 작고 집중된 변경을 선호한다.
- 중요한 도구나 프레임워크 선택 이유는 결과 보고 시 한국어로 짧게 설명한다.
- 파일을 수정하기 전 기존 프로젝트 구조를 확인하고, 관련 없는 사용자 변경을 덮어쓰지 않는다.
- Flutter 도구를 사용할 수 없는 경우 차단 사항을 문서화하고, 앱 생성이나 테스트가 완료된 것처럼 보고하지 않는다.
